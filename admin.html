🗑 Danger Zone

Clearing data will reset the portfolio to its original hardcoded content. This cannot be undone. Export a backup first!

📌 Follow these steps exactly once to get your portfolio live on GitHub Pages. After that, updating is just: edit in admin → save → commit the two HTML files.
1

Create a GitHub Account (if you don't have one)

Go to github.com and sign up for a free account. Use a professional username — professors may see this URL.

2

Create a New Repository

Click New Repository → Name it Portfolio → Set to Public → Check "Add a README file" → Click Create Repository.

3

Upload Your Two Portfolio Files

In your new repository, click Add file → Upload files. Upload both:

• index.html — your main portfolio page

• admin.html — this CMS panel

Click Commit changes.

4

Enable GitHub Pages

In your repository → Click Settings tab → Scroll to Pages (left sidebar) → Under "Source" select Deploy from a branch → Branch: main → Folder: / (root) → Click Save.

5

Wait 2–3 Minutes, Then Visit Your Site

Your portfolio will be live at: https://YOUR-USERNAME.github.io/Portfolio/

Your admin panel will be at: https://YOUR-USERNAME.github.io/Portfolio/admin.html

⚠️ Bookmark the admin URL — this is where you manage your site.

6

Upload Your CV to Google Drive

1. Upload your CV PDF to Google Drive.
2. Right-click → Share → Anyone with the link → Copy link.
3. Go to Personal & Links tab in this admin → paste link in CV / Resume Link field → Save.
4. The Download CV button on your portfolio will now work.

7

Add Your Professional Photo

In your GitHub repository, upload your photo as photo.jpg. Then in the repository, edit index.html → find <!-- Replace with: → uncomment the img tag and delete the placeholder div. Commit the change.

8

Making Future Updates

Go to your admin panel URL → Edit the content → Click Save Changes.
Your changes save instantly to your browser's local storage. Anyone visiting your portfolio URL will see the updates immediately (on the same computer).

To make changes permanent across all devices: after saving in the admin, click Backup Data → download the JSON → go to GitHub repository → upload both index.html and the JSON backup file. The portfolio will use the locally saved data on each visitor's machine.

Best practice: Keep a backup of your data JSON file and update the GitHub repo whenever you make significant changes.
